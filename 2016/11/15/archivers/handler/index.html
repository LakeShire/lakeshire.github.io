Handler机制相关源码分析(Java层)

发表于 2016-11-15 | 分类于 Android

android的Handler机制涉及Java层和native层，先看比较简单的Java层

Native层分析参见这篇博客

Handler

Handler允许你发送和处理Message和与线程消息队列相关联的Runnable，每个Handler实例与一个单独的线程和其消息队列相关联，当你创建新的Handler时，它绑定到创建它的线程的消息队列上，然后它将会发送消息到那个队列，以及处理它们

有两种用法：

  • 调度消息在未来某个时间点执行
  • 将一个动作加入，在不同的线程中执行

调度消息的方法有：

  • post()
  • postAtTime()
  • postDelayed()
  • sendEmptyMessage()
  • sendMessage()
  • sendMessageAtTime()
  • sendMessageDelayed()

postXXX()允许你将Runnable加入队列，当他们被接收时消息队列会调用
sendMessageXXX()允许你将包含数据的Message加入队列，将会被Handler的handleMessage()处理

当你的应用新建了一个进程，它的主线程运行一个消息队列，管理顶层应用组件和窗口。你可以创建你自己的线程，通过Handler与主应用通信

public Handler(Callback callback, boolean async) {
    if (FIND_POTENTIAL_LEAKS) {
        final Class<? extends Handler> klass = getClass();
        if ((klass.isAnonymousClass() || klass.isMemberClass() || klass.isLocalClass()) && (klass.getModifiers() & Modifier.STATIC) == 0) {
            Log.w(TAG, "The following Handler class should be static or leaks might occur: " +
                klass.getCanonicalName());
        }
    }

    //  获得当前线程的Looper
    //  如果没有，就是喜闻乐见的在没有调用Lopper.prepare()的线程中创建Handler的异常
    mLooper = Looper.myLooper();
    if (mLooper == null) {
        throw new RuntimeException(
            "Can't create handler inside thread that has not called Looper.prepare()");
    }

    mQueue = mLooper.mQueue;
    //  构造时候提供的一个全局使用的回调
    mCallback = callback;
    mAsynchronous = async;
}

//  这个Callback就是用来处理消息的
public interface Callback {
    public boolean handleMessage(Message msg);
}

分发消息

public void dispatchMessage(Message msg) {
    //  检查消息中的回调，回调中描述了怎样处理消息
    if (msg.callback != null) {
        //  使用消息中的回调
        handleCallback(msg);
    } else {
        //  使用全局提供的回调
        if (mCallback != null) {
            if (mCallback.handleMessage(msg)) {
                return;
            }
        }
        //  其他处理情况
        //  默认是个空实现，需要由子类重写
        handleMessage(msg);
    }
}

private static void handleCallback(Message message) {
    message.callback.run();
}

获得消息 

public final Message obtainMessage() {
    return Message.obtain(this);
}

一组obtainMessage()方法，都调用了Message的不同参数的obtain()方法

发送消息

public final boolean sendMessage(Message msg) {
    return sendMessageDelayed(msg, 0);
}

public final boolean sendMessageDelayed(Message msg, long delayMillis) {
    if (delayMillis < 0) {
        delayMillis = 0;
    }
    return sendMessageAtTime(msg, SystemClock.uptimeMillis() + delayMillis);
}

public boolean sendMessageAtTime(Message msg, long uptimeMillis) {
    MessageQueue queue = mQueue;
    if (queue == null) {
        RuntimeException e = new RuntimeException(this + " sendMessageAtTime() called with no mQueue");
        Log.w("Looper", e.getMessage(), e);
        return false;
    }
    return enqueueMessage(queue, msg, uptimeMillis);
}

private boolean enqueueMessage(MessageQueue queue, Message msg, long uptimeMillis) {
    //  把消息的目标设置为自己
    msg.target = this;
    if (mAsynchronous) {
        msg.setAsynchronous(true);
    }
    //  调用MessageQueue的消息入队方法
    return queue.enqueueMessage(msg, uptimeMillis);
}

post()

public final boolean post(Runnable r) {
   return  sendMessageDelayed(getPostMessage(r), 0);
}

private static Message getPostMessage(Runnable r) {
    Message m = Message.obtain();
    m.callback = r;
    return m;
}

其实也就是便利的sendMessageDelayed()，提供的Runnable作为回调

Looper

用来运行线程的消息循环的类，线程默认没有消息循环，这样来准备和开始一个消息循环

Looper.prepare();
......
Looper.loop();

大多数消息循环的交互是通过Handler类

public static void prepare() {
    prepare(true);
}

private static void prepare(boolean quitAllowed) {
    //  先从一个ThreadLocal中获得，如果已存在则抛异常，一个线程只能有一个Looper
    if (sThreadLocal.get() != null) {
        throw new RuntimeException("Only one Looper may be created per thread");
    }
    //  没有的话那么构造一个设置一下
    sThreadLocal.set(new Looper(quitAllowed));
}

//  私有的，不直接掉
private Looper(boolean quitAllowed) {
    //  构造一个消息队列
    mQueue = new MessageQueue(quitAllowed);
    //  设置为当前线程
    mThread = Thread.currentThread();
}

在prepare()中构造消息循环

ThreadLocal变量的使用，因为是个static方法，不同线程需要使用其独有的一份

public static void loop() {

    //  获得当前线程的Looper
    final Looper me = myLooper();

    //  要先调用prepare()构造Looper，否则抛异常
    if (me == null) {
        throw new RuntimeException("No Looper; Looper.prepare() wasn't called on this thread.");
    }

    final MessageQueue queue = me.mQueue;

    Binder.clearCallingIdentity();
    final long ident = Binder.clearCallingIdentity();

    //  开始一个无限循环
    for (;;) {
        //  取出消息队列中的下一个消息，可能阻塞
        Message msg = queue.next();
        //  消息为空就退出
        if (msg == null) {
            return;
        }

        Printer logging = me.mLogging;
        if (logging != null) {
            logging.println(">>>>> Dispatching to " + msg.target + " " + msg.callback + ": " + msg.what);
        }

        //  获得消息的目标Handler，分发消息让Handler处理
        msg.target.dispatchMessage(msg);

        if (logging != null) {
            logging.println("<<<<< Finished to " + msg.target + " " + msg.callback);
        }

        final long newIdent = Binder.clearCallingIdentity();
        if (ident != newIdent) {
            Log.wtf(TAG, "Thread identity changed from 0x" + Long.toHexString(ident) + " to 0x" + Long.toHexString(newIdent) + " while dispatching to " + msg.target.getClass().getName() + " " + msg.callback + " what=" + msg.what);
        }

        msg.recycleUnchecked();
    }
}

MessageQueue

是一个比较底层的类，包含一些native方法，持有被Looper分发的消息列表，消息不直接加入消息队列，而是通过与Looper相关联的Handler

可以通过Looper.myQueue()获得当前线程的消息队列

取下一个消息
当前消息队列为空的时候会休眠，让出CPU，在得到新消息后被唤醒，这样主线程就不会一直被死循环耗尽CPU，可以干别的事，这部分是在native代码中做的

Message next() {
    //  消息循环已经退出
    final long ptr = mPtr;
    if (ptr == 0) {
        return null;
    }

    int pendingIdleHandlerCount = -1;
    int nextPollTimeoutMillis = 0;
    for (;;) {
        if (nextPollTimeoutMillis != 0) {
            Binder.flushPendingCommands();
        }

        //  执行完毕或者超时才会返回
        nativePollOnce(ptr, nextPollTimeoutMillis);

        synchronized (this) {
            // 获得下一条消息 如果获得就退出
            final long now = SystemClock.uptimeMillis();
            Message prevMsg = null;

            //  这个mMessage是在加入队列的时候得到的
            Message msg = mMessages;
            if (msg != null && msg.target == null) {
                //  寻找下一条异步消息
                do {
                    prevMsg = msg;
                    msg = msg.next;
                } while (msg != null && !msg.isAsynchronous());
            }
            if (msg != null) {
                if (now < msg.when) {
                    //  下一条消息未就绪 设置一个超时就绪时唤醒
                    nextPollTimeoutMillis = (int) Math.min(msg.when - now, Integer.MAX_VALUE);
                } else {
                    //  获得一下消息 更新一下消息链表结构
                    mBlocked = false;
                    if (prevMsg != null) {
                        prevMsg.next = msg.next;
                    } else {
                        mMessages = msg.next;
                    }
                    msg.next = null;
                    if (DEBUG) Log.v(TAG, "Returning message: " + msg);

                    //  标记消息在使用中并返回
                    msg.markInUse();
                    return msg;
                }
            } else {
                // 没有更多消息
                nextPollTimeoutMillis = -1;
            }

            //  所有消息都被处理了，处理退出消息
            if (mQuitting) {
                dispose();
                return null;
            }

            //  空闲Handler
            //  第一次到这里，pendingIdleHandlerCount是-1，会被赋值
            if (pendingIdleHandlerCount < 0 && (mMessages == null || now < mMessages.when)) {
                pendingIdleHandlerCount = mIdleHandlers.size();
            }

            //  没有空闲Handler，等待
            if (pendingIdleHandlerCount <= 0) {
                mBlocked = true;
                continue;
            }

            if (mPendingIdleHandlers == null) {
                mPendingIdleHandlers = new IdleHandler[Math.max(pendingIdleHandlerCount, 4)];
            }
            mPendingIdleHandlers = mIdleHandlers.toArray(mPendingIdleHandlers);
        }

        //  运行空闲Handler 只在第一次迭代的时候
        for (int i = 0; i < pendingIdleHandlerCount; i++) {
            final IdleHandler idler = mPendingIdleHandlers[i];
            mPendingIdleHandlers[i] = null;

            boolean keep = false;
            try {
                keep = idler.queueIdle();
            } catch (Throwable t) {
                Log.wtf(TAG, "IdleHandler threw exception", t);
            }

            if (!keep) {
                synchronized (this) {
                    mIdleHandlers.remove(idler);
                }
            }
        }

        //  设置空闲Handler的数量为0，不再运行
        pendingIdleHandlerCount = 0;

        //  调用空闲Handler的时候，一条新消息可能到达，所以不用等待回去看一看挂起的消息

        nextPollTimeoutMillis = 0;
    }
}

添加消息进队列

boolean enqueueMessage(Message msg, long when) {
    if (msg.target == null) {
        throw new IllegalArgumentException("Message must have a target.");
    }

    if (msg.isInUse()) {
        throw new IllegalStateException(msg + " This message is already in use.");
    }

    synchronized (this) {
        if (mQuitting) {
            IllegalStateException e = new IllegalStateException(msg.target + " sending message to a Handler on a dead thread");
            Log.w(TAG, e.getMessage(), e);
            msg.recycle();
            return false;
        }

        msg.markInUse();
        msg.when = when;
        Message p = mMessages;
        boolean needWake;
        if (p == null || when == 0 || when < p.when) {
            //  原来的消息队列为空 或者需要立刻执行
            //  那么得到新的队列头
            //  如果阻塞了则唤醒队列
            msg.next = p;
            mMessages = msg;
            needWake = mBlocked;
        } else {
            //  消息加入队列
            needWake = mBlocked && p.target == null && msg.isAsynchronous();
            Message prev;
            for (;;) {
                prev = p;
                p = p.next;
                if (p == null || when < p.when) {
                    break;
                }
                if (needWake && p.isAsynchronous()) {
                    needWake = false;
                }
            }
            msg.next = p;
            prev.next = msg;
        }

        //  native方法唤醒
        if (needWake) {
            nativeWake(mPtr);
        }
    }
    return true;
}

Message

定义了一个消息，包含了描述和可被发送给Handler的数据对象，这个对象包含两个额外的int域和一个额外的object域，允许你在很多场合不用做内存分配

//  消息ID
public int what;
public int arg1; 
public int arg2;
public Object obj;
//  消息回复
public Messenger replyTo;
//  标记，比如是否正在使用
int flags;
//  执行时间
long when;
Bundle data;
//  目标Handler
Handler target;
Runnable callback;
Message next;

尽管构造函数是public的，最好的方式是obtain()或者obtainMessage()，将从一个回收对象的池中获得对象

public static Message obtain() {
    synchronized (sPoolSync) {
        //  池中有对象
        if (sPool != null) {
            Message m = sPool;
            sPool = m.next;
            m.next = null;
            m.flags = 0;
            sPoolSize--;
            return m;
        }
    }
    //  没有的话构造一个
    return new Message();
}

Message里面有一个next的Message域，整个就是一个链式结构
所以sPool也是一个消息池的链，在从池中取出sPool这个消息时，sPool本身被赋值为sPool.next，池大小减1

private static final int MAX_POOL_SIZE = 50;

这个定义了最大的池大小

public void recycle() {
    if (isInUse()) {
        if (gCheckRecycle) {
            throw new IllegalStateException("This message cannot be recycled because it "
                    + "is still in use.");
        }
        return;
    }
    recycleUnchecked();
}

void recycleUnchecked() {
    // Mark the message as in use while it remains in the recycled object pool.
    // Clear out all other details.
    flags = FLAG_IN_USE;
    what = 0;
    arg1 = 0;
    arg2 = 0;
    obj = null;
    replyTo = null;
    sendingUid = -1;
    when = 0;
    target = null;
    callback = null;
    data = null;

    synchronized (sPoolSync) {
        if (sPoolSize < MAX_POOL_SIZE) {
            next = sPool;
            sPool = this;
            sPoolSize++;
        }
    }
}

消息回收，就是把当前Message清空添加到sPool头部

总结

工作线程无法更新UI线程，或者需要延迟运行一段代码的时候，使用Handler发送消息给MessageQueue，线程中的Looper无限循环（主线程已经创建好了，自己的线程需要自己创建Looper），从MessageQueue从获得消息，交由对应的Handler处理

Activity中的便捷方法

public final void runOnUiThread(Runnable action) {
    if (Thread.currentThread() != mUiThread) {
        mHandler.post(action);
    } else {
        action.run();
    }
}

会查看是不是UI线程，如果不是就post()，是的话直接run()
